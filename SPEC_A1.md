## 0. このドキュメントの目的

審決起案「手続の経緯」の **1 行目（冒頭文）** を自動生成するロジックのうち、
**分割出願で原出願が PCT 国内段階の場合** の処理について、

- どの情報を、どこから取って、
- 何を条件にどの分岐に入り、
- 何という文を出力するか

を、コードを読まなくても追跡できるように記述する。

---

## 1. 前提：扱う案件類型

「分割_PCT原出願」とは次の構造を持つ出願:

```
原出願 (PCT)        ←  国際出願日に外国/日本特許庁等へ国際出願（出願人視点）
   ↓ 国内移行 (国内段階)
親出願 (PCT国内段階) ←  日本国内で手続継続。願書には「PCT国際出願番号」と
                       「国際出願日」が記載される
   ↓ 特許法44条1項により分割
本願 (分割出願)      ←  審判の対象
```

本願 1 件あたり、最大 3 階層を扱う：

- 第1世代：親 1 つ。親が PCT 国内段階
- 第2世代：親と祖父の 2 階層。**祖父**が PCT 国内段階
- 第3世代以降：A-1 のスコープ外（別パターン「分割_第3世代以降」）

## 2. 情報源と取得フィールド

### 2.1 メインデータ：`doc_history.json`

**情報源**: 特許庁 JPO API の「経過情報」レスポンス。
本願の出願番号 1 件で照会、`inventory/doc_history_collected/{appno}.json` に保存される。

A-1 修正で参照するフィールド：

| フィールドパス | 意味 | 用途 |
|---|---|---|
| `applicationNumber` | 本願出願番号（10桁） | テンプレに直接使用 |
| `filingDate` | 本願の出願日 | 「に新たな○○として」の日付 |
| `parentApplicationInformation.parentApplicationNumber` | 親出願番号（10桁） | 親 PCT 国内段階の特定 |
| `parentApplicationInformation.filingDate` | 親の出願日 | 「を国際出願日とする」の日付 |
| `divisionalApplicationInformation[].divisionalGeneration` | 各祖先の世代番号（0 = 最先） | 多世代分割の特定 |
| `divisionalApplicationInformation[].applicationNumber` | 各世代の出願番号 | 最先の出願番号判定 |
| `divisionalApplicationInformation[].internationalApplicationNumber` | 各世代の国際出願番号 | **言語判定（後述）** |
| `internationalApplicationNumber` | 本願自身の国際出願番号 | 本願が PCT 国内段階かの判定 |
| `bibliographyInformation[].documentList[].documentCode` | 提出/作成書類のコード | 翻訳文提出の有無判定 |
| `bibliographyInformation[].documentList[].documentDescription` | 提出/作成書類の名称 | 翻訳文提出の有無判定 |
| `priorityRightInformation[].parisPriorityDate` | パリ条約優先権の最先日 | 「（パリ条約による優先権主張　…）」 |
| `priorityRightInformation[].parisPriorityCountryCd` | 優先権の受理国（国コード） | 「米国」「欧州特許庁」等 |

### 2.2 補助データ：`aux_appinfo/{appno}.json`

**情報源**: 補助ソースを参照、
本願出願番号で照会、`inventory/aux_appinfo/{appno}.json` に保存される。

第2世代以降で **最先（祖父）の出願日** を取るために使用する。
理由: `divisionalApplicationInformation` の最先エントリには `filingDate` フィールドが
データソース仕様上含まれないため、補助ソースから取得する。

データ構造:
```json
{
  "chain": [
    {"appno": "2021168340", "filing_date": "2021-10-13", ...},   ← 本願
    {"appno": "2020087874", "filing_date": "2020-05-20", ...},   ← 親
    {"appno": "2018557229", "filing_date": "2017-02-20", ...}    ← 祖父（最先）
  ]
}
```

`chain[-1]`（配列の末尾）が最先の出願。

## 3. 分岐ロジック（分類器）

### 3.1 全体フロー（A-1 関連のみ抜粋）

```
本願の doc_history.json を読む
  │
  ├─ 親出願情報あり？
  │    │
  │    ├─ Yes (= 分割出願)
  │    │     │
  │    │     ├─ 世代 ≥ 3？  → 「分割_第3世代以降」（A-1 スコープ外）
  │    │     │
  │    │     ├─ 世代 = 2 で 最先がPCT国内段階？
  │    │     │   → 「分割_PCT原出願_第2世代」 ★ 新設
  │    │     │
  │    │     ├─ 世代 = 2 → 「分割_第2世代」
  │    │     │
  │    │     ├─ 親がPCT国内段階？ または 最先がPCT国内段階？
  │    │     │   → 「分割_PCT原出願」 ★ 汎用化
  │    │     │
  │    │     └─ それ以外 → 「分割_基本形」
  │    │
  │    └─ No → 別の判定経路（パリ条約優先権／PCT/国内優先権等）
  │
  └─ A-1 修正前：分割案件の最後に
     「apptype=C かつ パリ条約優先あり」で
     上書きされて「パリ条約優先＋外国語書面出願」に化けていた
     →修正後：分割案件には適用しない（バグ修正）
```

### 3.2 「PCT国内段階」の判定

出願番号 10 桁のうち **5 桁目が "5"** なら PCT 国内段階（先行検証で precision 100%）。

例:
- `2022510315` → 5 桁目 "5" → PCT 国内段階
- `2018244177` → 5 桁目 "2" → 通常出願

ロジック: `classify.py:is_pct_national_phase()`

### 3.3 「最先がPCT国内段階」の判定（第2世代対応）

第2世代分割で「祖父」が PCT かを判定する 2 通りの方法を併用：

| ロジック名 | 情報源 | 判定 |
|---|---|---|
| `chain_root_is_pct_national_phase()` | `doc_history.json` の `divisionalApplicationInformation` で `divisionalGeneration=0` のエントリ | その `applicationNumber` の 5 桁目 = "5"、または `internationalApplicationNumber` が空でない |
| `aux_root_is_pct_national_phase()` | 補助ソース由来の `chain[-1].appno` | 同上の 5 桁目判定 |

両者を **or** で結合。どちらかで PCT 国内段階が確認できれば true。

## 4. 出力テンプレート

### 4.1 「分割_PCT原出願」（第1世代）

```
本願は、{祖父の国際出願日}を国際出願日とする{言語フレーズ}（特願{親出願番号}号）の
一部を、{本願の出願日}に新たな{終端フレーズ}としたものであって
（パリ条約による優先権主張　{パリ優先情報}）、
その手続の経緯の概略は、次のとおりである。
```

※ `（パリ条約による優先権主張…）` の括弧はパリ条約優先権が無ければ省略。

#### 4.1.1 `{言語フレーズ}` の決定（apptype_phrase）

`divisionalApplicationInformation` の `divisionalGeneration=0` エントリの
`internationalApplicationNumber` 接頭辞で判定。

| 接頭辞 | 表記 | 法的根拠 |
|---|---|---|
| `JP*` | 日本語特許出願 | 特許法 184 条の 6 第 1 項（日本語 PCT） |
| 上記以外（`US*` `EP*` `IB*` `CN*` `KR*` 等） | 外国語特許出願 | 特許法 184 条の 4 第 1 項（外国語 PCT） |
| 空 | 特許出願 | 情報不足時の安全側フォールバック |

ロジック: `classify.py:divisional_root_apptype_phrase()`

#### 4.1.2 `{終端フレーズ}` の決定（terminal_phrase）

本願自身が翻訳文を提出していれば「外国語書面出願」、
していなければ「特許出願」。

判定: `bibliographyInformation[].documentList[]` のうち以下のいずれかを含むか：

- `documentCode` が `A632`
- `documentDescription` に「翻訳文」を含む（実例: `A631` の `documentDescription` が "翻訳文提出書"）

| 翻訳文 | 表記 | 法的位置付け |
|---|---|---|
| あり | 外国語書面出願 | 184 条の 4 で提出された翻訳文に基づく取扱い |
| なし | 特許出願 | 翻訳文不要案件（日本語 PCT 等） |

ロジック: `classify.py:divisional_terminal_phrase()`

#### 4.1.3 実例

| 案件 | apptype_phrase | terminal_phrase | 出力例（先頭部分） |
|---|---|---|---|
| 外国語 PCT + 翻訳文あり | 外国語特許出願 | 外国語書面出願 | 「…外国語特許出願（特願2020-524880号）の一部を…新たな外国語書面出願として…」 |
| **日本語 PCT** | 日本語特許出願 | 特許出願 | 「…**日本語特許出願**（特願2019-567110号）の一部を…新たな**特許出願**として…」 |
| 外国語 PCT + 翻訳文なし | 外国語特許出願 | 特許出願 | 「…外国語特許出願（特願2020-511862号）の一部を…新たな**特許出願**として…」 |

### 4.2 「分割_PCT原出願_第2世代」（新設）

```
本願は、{祖父の国際出願日}を国際出願日とする{言語フレーズ}（特願{祖父出願番号}号）の
一部を、{親の出願日}に新たな特許出願とした特願{親出願番号}号の一部を、
{本願の出願日}に新たな{終端フレーズ}としたものであって
（パリ条約による優先権主張　{パリ優先情報}）、
その手続の経緯の概略は、次のとおりである。
```

第1世代との違い:
- 出願チェーンが 3 段（祖父→親→本願）
- 中間の「親」を「に新たな特許出願とした特願{親出願番号}号」として明示
- 祖父の出願番号は `divisionalApplicationInformation[gen=0]` から
- 祖父の出願日は 補助ソース `chain[-1].filing_date` から（前述 §2.2 の理由）

唯一の実例は `2023-020552`（祖父が WIPO 国際局受理 = `IB*` 始まりの国際出願番号）。

## 6. ロジックの根拠（条文・基準・開発者ルール）

| 判定 | 根拠 |
|---|---|
| PCT 国内段階の判定（5桁目=5） | 出願番号付番ルール（A1_verify_pct_5kt_rule.py で precision 100% 検証済） |
| 日本語 PCT の表記 | 特許法 184 条の 6 第 1 項（日本語 PCT 出願は翻訳文不要） |
| 外国語 PCT の表記 | 特許法 184 条の 4 第 1 項（外国語 PCT 出願は翻訳文提出義務） |
| 分割の表記 | 特許法 44 条 1 項 |
| 「パリ条約による優先権主張」 | パリ条約第 4 条／特許法 43 条 |
| 「同年」「同月」省略の扱い | 1 部門ローカルルール（`knowledge/rules/_comments_extracted.txt`） |

## 7. 既知の許容差（裁量例外、変更しない）

corpus と比較したときに残る差異のうち、起案者裁量と判断したもの:

- 行頭全角空白の有無（テンプレでは「　本願は、」と全角空白入り）
- 西暦と和暦の併記（テンプレでは「2017 年（平成 29 年）」併記）
- 出願番号を括弧で囲む／囲まない
- 「特許法 44 条 1 項の規定に基づいて」の挿入有無（起案者裁量）
- 優先日定義行の挿入位置（テンプレでは「次のとおりである。なお、…」末尾に統一）
- 「IB*（WIPO 国際局受理）」での「特許出願」中立表記（テンプレは「外国語特許出願」を採用）

詳細は `inventory/_diff_dive_A1.md` §B を参照。

## 10. 用語集

| 用語 | 意味 |
|---|---|
| **本願** | 審判の対象出願 |
| **親出願 / 祖父出願** | 分割の元になった出願（直接の親が「親」、その親が「祖父」） |
| **PCT 国内段階** | PCT 国際出願を日本国内に移行した出願（出願番号 5 桁目が "5"） |
| **外国語 PCT 出願** | PCT 国際出願を外国語で行ったもの。日本移行時に翻訳文提出が必要（184 条の 4） |
| **日本語 PCT 出願** | PCT 国際出願を日本語で行ったもの。翻訳文は不要（184 条の 6） |
| **外国語書面出願** | 特許法 36 条の 2 第 1 項に基づき外国語で願書を提出した出願 |
| **分割出願** | 特許法 44 条 1 項に基づき出願の一部を新たな出願として分けたもの |
| **パリ条約優先権** | パリ条約第 4 条に基づき、外国出願を基礎とする優先権主張（特許法 43 条） |
| **国内優先権** | 日本国内の先願を基礎とする優先権主張（特許法 41 条） |
| **doc_history.json** | 特許庁 JPO API から取得した経過情報の生 JSON |
| **corpus** | 確定済み起案文を抽出して保存したもの。比較の基準 |
| **pattern** | 「分割_PCT原出願」「パリ条約優先＋外国語書面出願」等の冒頭文の類型名 |
| **ratio** | 生成文と corpus の文字列一致率（`difflib.SequenceMatcher`） |
